# 第一周 WMS 登录与菜单标签联动设计

## 背景

第一周课上作业要求自主完成 `Vue3 + Element Plus + Spring Boot` 的登录打通，并开发一个菜单栏和标签页联动界面。老师后续澄清：这是每周作业，按小组评分，代码和截图需要上传到作业系统，周四上课每组派代表简单介绍开发逻辑，不需要 PPT。

课程整体方向是汽车物流信息系统开发实训，后续课程会围绕小型仓储管理系统展开。第一周作业不做完整 WMS，只做一个可演示、可截图、可讲清逻辑的后台雏形。

## 范围

本周实现以下内容：

- 前端使用 `Vue3 + Vite + Element Plus`。
- 后端使用 `Spring Boot`。
- 登录页调用后端登录接口，完成真实前后端请求打通。
- 登录成功后进入后台主界面。
- 后台主界面包含左侧菜单栏和顶部标签页。
- 点击菜单时，打开或切换对应标签页。
- 标签页可关闭，关闭当前页后自动切换到相邻标签。
- 业务页面使用 WMS 方向的占位页，便于课堂介绍和截图。

本周暂不实现以下内容：

- 不接 MySQL。
- 不做真实物料、入库、出库、库存业务 CRUD。
- 不做权限角色体系。
- 不做 JWT 的完整签名和刷新机制。
- 不做移动端适配。

## 技术方案

### 前端

前端项目放在 `frontend/`。

核心依赖：

- `vue`
- `vite`
- `element-plus`
- `vue-router`
- `pinia`
- `axios`

主要页面：

- `/login`：登录页。
- `/`：后台布局页，未登录访问时跳转到登录页。
- `/dashboard`：首页。
- `/materials`：物料信息。
- `/inbound`：入库管理。
- `/inventory`：库存监控。
- `/outbound`：出库管理。

主要组件：

- `LoginView`：登录表单、错误提示、登录请求。
- `MainLayout`：后台整体布局。
- `SideMenu`：左侧菜单。
- `TabBar`：顶部标签页。
- `PlaceholderPage` 或独立业务占位页：展示每个 WMS 模块的标题和简短说明。

状态管理：

- `auth` store 保存 token 和用户信息。
- `tabs` store 保存已打开标签、当前激活标签。
- token 可保存到 `localStorage`，刷新页面后能保持登录状态。

### 后端

后端项目放在 `backend/`。

核心依赖：

- `Spring Boot Web`
- `Spring Boot Validation`

接口：

- `POST /api/auth/login`
  - 入参：`username`、`password`
  - 成功：返回 token、用户名、显示名。
  - 失败：返回 401 和错误信息。
- `GET /api/auth/me`
  - 入参：请求头 `Authorization: Bearer <token>`
  - 成功：返回当前用户信息。
  - 失败：返回 401。

账号策略：

- 内存固定账号：`admin / 123456`。
- 登录成功返回演示用 token，例如 `demo-token-admin`。
- 本周重点是接口打通，不做数据库用户表。

跨域：

- 后端允许前端开发地址 `http://localhost:5173` 访问。

## 交互逻辑

登录流程：

1. 用户访问前端。
2. 未登录时进入 `/login`。
3. 输入 `admin / 123456`。
4. 前端调用 `POST /api/auth/login`。
5. 后端校验成功后返回 token。
6. 前端保存 token 和用户信息。
7. 跳转到后台首页。

菜单和标签页联动：

1. 用户点击左侧菜单。
2. 前端根据菜单配置找到路由和标题。
3. 如果对应标签不存在，则新增标签。
4. 如果标签已存在，则直接切换到该标签。
5. 路由同步跳转到对应页面。
6. 用户关闭标签时，从标签列表移除。
7. 如果关闭的是当前标签，则切换到左侧或右侧相邻标签。
8. 首页标签默认打开，建议不可关闭，避免关闭全部标签后界面空白。

## 页面结构

后台主界面采用常见管理系统结构：

- 顶部：系统名、当前用户、退出按钮。
- 左侧：模块菜单。
- 中间顶部：标签页栏。
- 主区域：当前标签对应页面内容。

菜单项使用 WMS 课程方向命名：

- 首页
- 物料信息
- 入库管理
- 库存监控
- 出库管理

每个占位页展示模块名称、用途说明和少量示例字段，方便截图和课堂说明。

## 错误处理

- 用户名或密码为空时，前端表单校验提示。
- 密码错误时，后端返回 401，前端提示“用户名或密码错误”。
- 未登录访问后台路由时，前端跳转到登录页。
- token 无效时，前端清理登录状态并返回登录页。
- 登录请求失败时，前端显示网络或服务异常提示。

## 验收标准

本周作业完成后应满足：

- 后端服务可在 `http://localhost:8080` 启动。
- 前端服务可在 `http://localhost:5173` 启动。
- 输入错误账号时有错误提示。
- 输入 `admin / 123456` 可进入后台。
- 登录接口能在浏览器开发者工具中看到真实请求。
- 点击左侧菜单可以新增或切换顶部标签。
- 关闭标签后界面状态正确。
- 退出登录后返回登录页。
- 可提交代码和关键截图。

- 验证错误账号登录失败。

- 启动 Vite 开发服务。
- 手动验证登录成功和失败。
- 手动验证菜单点击、标签新增、标签切换、标签关闭。
- 手动验证刷新后登录态保留。
- 手动验证退出登录。

集成：

- 同时启动前端和后端。
- 浏览器访问 `http://localhost:5173` 完成完整流程。
- 截取登录页、后台首页、菜单标签联动页面、登录请求或接口返回截图。

## 课堂介绍要点

代表介绍时可以按这个顺序讲：

1. 前后端分离：前端负责页面和交互，后端负责登录校验。
2. 登录打通：前端用 axios 调用 Spring Boot 登录接口，成功后保存 token。
3. 路由守卫：未登录不能进入后台。
4. 菜单配置：菜单项和路由统一配置。
5. 标签联动：点击菜单时根据路由判断新增还是切换标签。
6. 本周边界：先用内存账号完成流程，后续课程再扩展数据库和业务模块。
